📖 Справка по режимам шифрования

Режимы шифрования:

  • AES-256-GCM — надёжное шифрование паролем. Результат — строка Base64.
  • Невидимые пробелы — AES + спец. символы (zero-width spaces). Текст выглядит как обычный.
  • Base64 — AES + стандартное кодирование Base64.
  • Компрессия — AES + сжатие gzip. Укорачивает длинные сообщения.
  • Эмодзи — AES + эмодзи. Выглядит как набор смайлов.
  • Китайские — AES + китайские иероглифы.
  • Раскладка — замена раскладки клавиатуры (EN↔RU). Без шифрования, только обфускация.

Направление: «Шифровать» → кодирует, «Дешифровать» → раскодирует. Расшифровка автоматически определяет тип кодировки.

Рекомендации: Для максимальной безопасности используйте AES-256-GCM с длинным паролем. Для стеганографии — см. вкладку «Стего».

📖 Справка по стеганографии

Как работает стеганография: Секретное сообщение шифруется и «вшивается» в текст-носитель. Кодирование и декодирование работают БЕЗ оригинала — только стего-текст и пароль.

Лингвистические каналы (авто-определяются): синонимы, аббревиатуры, пунктуация, даты, ё/е, пробелы, телефоны, URL, email, ФИО, эмодзи и др.

Теговые каналы (вставляются через алиасы в текст):

  • [steg-fio] — российские ФИО (33 бит)
  • [steg-pc-proc], [steg-pc-ram], [steg-auto-engine], [steg-gadget-phone] — комплектующие, гаджеты и др.
  • [steg-recipe-universal], [steg-recipe-meat] — рецепты (117 бит/тег)
  • [steg-address] — адреса РФ с аббревиатурами как доп. биты (52 бит/тег)
  • [stego-music] — музыкальные треки, 8192×8192 комбинаций (26 бит/тег)
  • [steg-code-python], [steg-code-typescript], [steg-code-go], [steg-code-rust], [steg-code-css] — полноценные файлы кода (256 бит/тег)
  • [steg-json] — JSON конфиги с UUID — 128 бит/тег, самый ёмкий!
  • [steg-movie] — фильмы в кавычках, ~45000 названий (≤15 бит/вхождение)
  • [steg-videogame] — видеоигры, ~800 названий (≤10 бит/вхождение)
  • [steg-dog], [steg-cat] — породы собак/кошек (≤9 бит/вхождение)

Подсказка: Введите `[steg-` в поле текста-носителя — автодополнение покажет все доступные алиасы.

Важно: Не все сообщения можно зашифровать в любой текст. Ёмкость зависит от длины и содержания носителя. Используйте длинные тексты с алиасами для максимальной ёмкости.

📖 Справка по экстремальным методам

Экстремальные методы кодируют данные напрямую в текст, не используя лингвистические каналы. Каждый метод работает независимо и может быть включён/выключен.

  • Zero-width символы — невидимые символы между буквами. Самый ёмкий метод (~2 бит/символ).
  • Пробелы — замена обычных пробелов на неразрывные. 1 бит/пробел.
  • Регистр — чередование верхнего/нижнего регистра. 1 бит/буква.
  • Кириллица → Латиница — замена похожих букв (А→A, О→O и т.д.). 1 бит/позиция.

Автообнаружение: При декодировании методы определяются автоматически. Пароль нужен всегда.

⚠ Предупреждение: Все методы заметны при внимательном просмотре текста.

📖 Справка по Марков тексту

Марков текст-стего — генерирует текст на основе цепей Маркова (n-грамм), встраивая секретное сообщение в выбор слов. Результат выглядит как естественный текст.

Корпус — набор текстов (например, переписки), из которого строится n-граммная модель. Чем больше корпус, тем естественнее результат. Загружается как .zip архив с .txt файлами.

Кодирование: секретное сообщение → стего-текст. Декодирование: стего-текст → секретное сообщение (нужен тот же корпус).

📖 Справка по LLM Стего

LLM Стего — стеганография на основе языковой модели. Скрытое сообщение встраивается в выбор токенов при генерации текста.

Backend — Python сервис (stego-v4) с GGUF моделью. Должен быть запущен на вашем ПК.

Ёмкость: ~1-2 байта на токен. Для 100 байт нужно ~50-100 токенов генерации.
